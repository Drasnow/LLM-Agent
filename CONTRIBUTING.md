# Contributing

感谢你愿意帮助维护这个仓库。

这个项目是一个面向中文开发者的 LLM / RAG / Agent 学习资源导航，欢迎通过补充资源、修复链接、纠正文案和优化分类的方式参与共建。

## 你可以贡献什么

- 新增优质学习资源
- 修复失效链接
- 补充资源简介、适合人群、推荐理由
- 调整错误分类
- 更新已经过时的信息
- 完善路线建议和资源导航

## 收录标准

原则上，新增资源应满足以下条件：

- 公开可访问
- 与 LLM、Prompt、RAG、Agent、Workflow 学习强相关
- 内容质量稳定，有实质学习价值
- 适合中文开发者，或属于高价值官方文档 / 国际主流教程

原则上不收录：

- 纯营销页、导流页
- 内容极少且没有明确价值的项目
- 长期失效、长期停止维护且没有明显参考价值的资源
- 与已有资源高度重复但质量更弱的项目

## 内容结构

当前主分类固定为：

- `LLM 基础`
- `Prompt / 应用开发`
- `RAG / 知识库`
- `Agent / Workflow`
- `课程 / 教程`
- `导航 / Awesome`

新增资源请优先提交到对应的 `resources/*.md` 文件中，而不是直接大幅修改 `README.md`。

## 提交格式

每条资源至少包含以下信息：

- 名称
- 链接
- 分类
- 简介
- 适合谁
- 推荐理由

建议使用下面的格式：

```md
### 资源名称

- 链接：https://example.com
- 分类：Agent / Workflow
- 简介：一句话介绍这个资源是什么。
- 适合谁：适合什么阶段、什么目标的人。
- 推荐理由：为什么值得收录。
```

## 推荐贡献流程

1. 先搜索仓库，确认资源没有被重复收录
2. 如果是新增资源，优先先开一个 Issue 说明价值
3. 如果是明显错误或失效链接，也可以直接提交 PR
4. 提交 PR 时说明你改了什么，以及为什么值得合并

## PR 检查清单

提交前请尽量确认：

- 链接可访问
- 分类合理
- 简介准确、简洁
- 没有和现有资源重复堆砌
- 没有把大量详细内容重新塞回 `README.md`

## 维护原则

- README 只做首页导航和精选入口
- 详细资源尽量放到 `resources/`
- 如果两个资源高度重复，优先保留更权威、更系统、更活跃的一项
- 维护者会根据质量和仓库定位决定是否合并

## 行为规范

参与贡献默认受 [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) 约束。
